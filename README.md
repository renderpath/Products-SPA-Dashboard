# Products SPA Dashboard

Products SPA Dashboard — учебное SPA-приложение на React для практики работы с современным frontend-стеком: React Router, Redux Toolkit, RTK Query, TypeScript и Webpack.

Приложение работает с DummyJSON API: получает товары, показывает детальную информацию, выполняет авторизацию пользователя и кэширует данные через RTK Query.

## Почему выбран такой стек

Проект собран вручную через Webpack без Create React App, чтобы показать понимание базовой настройки frontend-приложения: entry point, loaders, dev server, HTML template и production build.

Redux Toolkit используется для управления клиентским состоянием: авторизация, избранное, фильтры и пагинация.

RTK Query используется для работы с API, загрузки и кэширования данных.

React Router используется для маршрутизации между страницами и защиты приватных маршрутов.

## Установка и запуск

Склонировать репозиторий:

```bash
git clone https://github.com/renderpath/Products-SPA-Dashboard.git
```

Перейти в папку проекта:

```bash
cd Products-SPA-Dashboard
```

Установить зависимости:

```bash
npm install
```

Запустить проект в режиме разработки:

```bash
npm start
```

После запуска приложение будет доступно по адресу:

```txt
http://localhost:3000
```

## Работа с API

В проекте используется DummyJSON API.

Основные endpoints:

| Endpoint            | Назначение               |
| ------------------- | ------------------------ |
| `POST /auth/login`  | Авторизация пользователя |
| `GET /products`     | Получение списка товаров |
| `GET /products/:id` | Получение одного товара  |

Запросы реализованы через RTK Query в файле:

```txt
src/shared/api/dummyJsonApi.ts
```

RTK Query отвечает за:

* выполнение запросов;
* хранение состояния загрузки;
* обработку ошибок;
* кэширование данных;
* предоставление готовых hooks для компонентов.

## Управление состоянием

В проекте используется Redux Toolkit.

Состояние разделено на slices:

| Slice            | Назначение                                        |
| ---------------- | ------------------------------------------------- |
| `authSlice`      | Данные авторизованного пользователя               |
| `favoritesSlice` | ID товаров, добавленных в избранное               |
| `filtersSlice`   | Поиск, фильтрация, сортировка и пагинация товаров |
| `dummyJsonApi`   | API slice для RTK Query                           |

## Авторизация

Авторизация реализована через DummyJSON Auth.

После успешного входа данные пользователя сохраняются:

* в Redux;
* в localStorage.

Это позволяет не терять авторизацию после перезагрузки страницы.

Для выхода используется action `logout`, который очищает Redux-состояние и localStorage.

## Защищённые маршруты

Приватные страницы обёрнуты в компонент `ProtectedRoute`.

Если пользователь не авторизован, он перенаправляется на страницу входа:

```txt
/login
```

Защищённые страницы:

```txt
/products
/products/:productId
/favorites
/profile
```

## Валидация формы входа

На странице авторизации добавлена клиентская валидация.

Проверяется:

* заполненность username;
* минимальная длина username;
* отсутствие пробелов в username;
* заполненность password;
* минимальная длина password;
* отсутствие пробелов в password.

Ошибки отображаются под соответствующими полями.

Также используются атрибуты доступности:

* `aria-invalid`;
* `aria-describedby`.

## Каталог товаров

На странице `/products` реализованы:

* вывод списка товаров;
* поиск по названию, описанию и категории;
* фильтрация по категории;
* сортировка по цене и рейтингу;
* пагинация;
* добавление товара в избранное;
* переход на детальную страницу товара;
* состояния loading, error и empty state.

## Детальная страница товара

На странице `/products/:productId` отображается:

* изображение товара;
* название;
* категория;
* описание;
* цена;
* рейтинг;
* количество на складе;
* размер скидки;
* кнопка добавления или удаления из избранного.

## Избранное

На странице `/favorites` отображаются товары, добавленные пользователем в избранное.

Реализованы:

* поиск среди избранных товаров;
* фильтрация по категории;
* сортировка по цене и рейтингу;
* пагинация;
* удаление товара из избранного;
* empty state, если избранных товаров нет;
* empty state, если товары не найдены по текущим фильтрам.
